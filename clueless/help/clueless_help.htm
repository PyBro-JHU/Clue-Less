PyBro Clue-Less Game Help

 

Welcome to the PyBro Clue-Less Game.  This Help document provides an overview of system requirements, game rules, and common gameplay functionality. 

 

The PyBro Clue-Less Game can be played by any users connected to the Internet.  The PyBro Clue-Less Game server manages game play.  Each PyBro Clue-Less Game client can connect to the server via the Internet to participate in the game.

 

System Requirements

 

-       Any computer with a Python runtime environment.

-       Internet connectivity.

 

Rules

 

The PyBro Clue-Less game follows the rules of the classic Hasbro game of Clue with some modifications as described below.  The rules for Clue can be found here:

http://www.hasbro.com/common/instruct/clueins.pdf

 

The rules of the PyBro Clue-Less game follow the rules of the class Hasbro game of Clue, except for some exceptions when moving from room to room.  In particular, each hallway only holds one person. If someone is currently in a hallway, you may not move there.  When it is your turn, you do not need to roll a die.   If you are in a room, you may: (a) move through one of the doors to the hallway (if it is not blocked), (b) take a secret passage to a diagonally opposite room (if there is one) and make a suggestion, or (c) if you were moved to the room by another player making a suggestion, you may, if you wish, stay in that room and make a suggestion. Otherwise you may move through a doorway or take a secret passage as described above.   However, if you are in a hallway, you must move to one of the two rooms accessible from that hallway and make a suggestion. 

 

If all of the exits are blocked (i.e., there are people in all of the hallways) and you are not in one of the corner rooms (with a secret passage), and you weren’t moved to the room by another player making a suggestion, you lose your turn (except for maybe making an accusation).  Your first move must be to the hallway that is adjacent to your home square.  Whenever a suggestion is made, the room must be the room the one making the suggestion is currently in. The suspect and weapon in the suggestion are moved to the room in the suggestion.  You may make an accusation at any time during your turn.

 

 

 

Starting the Game

 

To start a PyBro Clue-Less game, open the PyBro Clue-Less game client by double-clicking on the game application icon or running the command “python hci.py.”

 

Initially, the PyBro Clue-Less game client displays a start screen.  The start screen prompts the user to enter a unique username that identifies the user.  You may type your username into the username field.  Additionally, the start screen provides a list of suspects that the user may choose, and the user may select a suspect as the user’s character for game play.

 

Upon entering a username and selecting a suspect, you may click the Play! button to initiate game play.  Clicking the Play! button will cause the PyBro Clue-Less game client to connect to the PyBro Clue-Less game server to begin game play.

 

Playing the Game

 

As further described below, the PyBro Clue-Less game client provides a main user interface with multiple regions.  The particular regions include: (1) Game Board (2) Notifications (3) Detective Notebook and (4) Game Play Actions.

 

(1) Game Board

 

The Game Board provides an interactive illustration of the various rooms and hallways that each suspect may move to during game play.  To facilitate game play, a user may move his or her suspect icon to different spaces on the Game Board in accordance with the Rules.  The suspect icon will be shown on the current hallway or room space for the suspect.  When it is a player’s turn, the play may move his or her suspect icon to a different hallway or room space. 

 

(2) Notifications

 

The Notifications region of the main user interface provides a visual display of new updates to game play.  As player make take actions during the game, such as moving, making accusations, and making suggestions, corresponding updates are displayed in the Notifications region.  For instance, if a suspect moves during the game, the Notifications region of the main user interface indicates the movement of the suspect.  During a player’s turn, if the player makes a suggestion, the details of the suggestion are output to the Notifications region.  Similarly, if a player makes an accusation, the results of the accusation are output to the Notifications region.  In this way, the Notifications region enables each player to receive continual updates of game play information based on the progression of the game.  Information that indicates players winning, losing, joining, or exiting the game are also displayed in the Notifications region.

 

(3)  Detective Notebook

 

The Detective Notebook lists the Weapons, Rooms, and Suspects that are included in the user’s initial deck of cards or that have been revealed to the user during game play.  As a player sees additional cards during game play, the Detective Notebook allows the user to deductively exclude possible Weapons, Rooms, and Suspects in order to win the game. 

 

(4) Game Play Actions

 

The main user interface also includes a region with Game Play Actions.  The Game Play Actions allow the user to make a Suggestion, Accusation, or End a Turn. 

 

To make a Suggestion the user may click the Suggestion button.  A new window will prompt the user to enter a Weapon, Suspect, and Room.  Upon submitting the combination of Weapon, Suspect, and Room, the other players in the game will receive a prompt to disprove the Suggestion by revealing a card.  In this way, the user may exclude items during game play. 

 

If the user wishes to make an Accusation, the user must be in the room in which the Accusation is made.  The user may click the Accusation button and then specify a Weapon and Suspect.  Upon submitting the combination of Weapon and System, the game will determine whether the combination of Weapon, Suspect, and Room is correct.  If the combination is correct, the user wins the game and all other players are notified. However, if the player is incorrect, the player loses the game and all other players are notified. 

 

Quitting the Game

 

To quick the game, the user may select the Quit button in the main user interface.
